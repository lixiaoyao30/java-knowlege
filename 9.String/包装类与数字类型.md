# 包装类与数字类型

---

## 一、什么是包装类

- Java 为 **8 种基本数据类型** 提供了对应的**类**（包装类）。
- 包装类本质是**引用数据类型**。
- 作用：让基本类型可以按"对象"来使用，便于放入集合、表达 `null`、调用对象方法等。

### 1.1 8 种基本类型与包装类对照

| 基本类型 | 包装类（java.lang） |
| --- | --- |
| `byte` | `Byte` |
| `short` | `Short` |
| `int` | `Integer` |
| `long` | `Long` |
| `float` | `Float` |
| `double` | `Double` |
| `boolean` | `Boolean` |
| `char` | `Character` |

**重点记忆：**
- 最常用：`Integer`
- 特殊拼写要记住：`int` → `Integer`，`char` → `Character`
- 包装类都在 `java.lang` 包下

### 1.2 6 个数字包装类继承 Number

`Byte`、`Short`、`Integer`、`Long`、`Float`、`Double` 都继承了 `Number` 类，因此都有：

- `byteValue()`、`shortValue()`、`intValue()`、`longValue()`、`floatValue()`、`doubleValue()`

这些方法的作用：**把包装类对象转回基本类型**（即**拆箱**）。

> `Boolean` 和 `Character` 不继承 `Number`，所以不在这 6 个数字类里。

## 二、装箱与拆箱

**装箱**：基本类型 → 包装类对象  
**拆箱**：包装类对象 → 基本类型

```java
public class IntegerTest02 {
    public static void main(String[] args) {
        // 装箱：int -> Integer
        int i = 100;
        Integer i1 = new Integer(i);    // 过时写法，不推荐
        Integer i2 = Integer.valueOf(i); // 推荐，有缓存
        Integer i3 = i;                  // 自动装箱，最常用

        // 拆箱：Integer -> int
        int num = i2.intValue();  // 手动拆箱
        int num2 = i2;            // 自动拆箱，最常用

        // Boolean 装箱
        boolean a = true;
        Boolean b = a;                    // 自动装箱
        Boolean c = Boolean.valueOf(a);   // 手动装箱

        System.out.println("b getClass(): " + b.getClass().getName());
        System.out.println("c getClass(): " + c.getClass().getName());
    }
}
```

**装箱方式总结：**

| 写法 | 说明 |
| --- | --- |
| `new Integer(100)` | 废弃，不要用 |
| `Integer.valueOf(100)` | 推荐，有缓存 |
| `Integer x = 100` | 自动装箱，最常用 |

**拆箱方式总结：**

| 写法 | 说明 |
| --- | --- |
| `int a = obj` | 自动拆箱，最常用 |
| `int a = obj.intValue()` | 手动拆箱，了解即可 |
| `null` 对象拆箱 | `NullPointerException`！先判空再拆箱 |

## 三、Integer 常用方法详解

### 静态方法速查

| 方法 | 说明 |
| --- | --- |
| `Integer.compare(x, y)` | 比较大小，返回 -1 / 0 / 1 |
| `Integer.max(x, y)` | 返回较大值 |
| `Integer.min(x, y)` | 返回较小值 |
| `Integer.parseInt("123")` | 字符串 → int 基本类型 ⭐ |
| `Integer.valueOf(100)` | int → Integer 对象（装箱，有缓存）⭐ |
| `Integer.valueOf("123")` | 字符串 → Integer 对象 |
| `Integer.toString(100)` | int → 字符串 |
| `Integer.toBinaryString(n)` | 十进制 → 二进制字符串 |
| `Integer.toHexString(n)` | 十进制 → 十六进制字符串 |
| `Integer.toOctalString(n)` | 十进制 → 八进制字符串 |
| `Integer.MAX_VALUE` | 最大值常量：2147483647 |
| `Integer.MIN_VALUE` | 最小值常量：-2147483648 |

### 实例方法速查

| 方法 | 说明 |
| --- | --- |
| `obj.compareTo(other)` | 比较两个 Integer 对象大小 |
| `obj.equals(other)` | 比较两个 Integer 对象的值 ⭐ |
| `obj.toString()` | Integer → 字符串 |
| `obj.intValue()` | Integer → int（手动拆箱） |

```java
public class IntegerTest03 {

    @Test
    public void testBoxing() {
        // int -> Integer（装箱）
        Integer i1 = Integer.valueOf(100);
        System.out.println(i1);
        // "100" -> Integer（字符串转对象）
        Integer i2 = Integer.valueOf("100");
        System.out.println(i2);
        // 注意: java.lang.NumberFormatException
        // Integer i3 = Integer.valueOf("abc");
    }

    @Test
    public void testIntegerToString() {
        // int -> String
        String str = Integer.toString(100);
        System.out.println(str);

        int i = 100;
        String s = i + "";  // 另一种写法
        System.out.println(s);
    }

    @Test
    public void testUnBoxing() {
        Integer i = new Integer(100); // 演示写法，new Integer 已过时
        int num = i.intValue(); // 拆箱
        System.out.println(num + 1);
    }

    // compare —— 静态方法，比较两个 int 大小
    // 规律：前 < 后 → 负数(-1)，前 > 后 → 正数(1)，相等 → 0
    @Test
    public void testCompare() {
        System.out.println(Integer.compare(10, 20)); // -1 (10 < 20)
        System.out.println(Integer.compare(20, 10)); //  1 (20 > 10)
        System.out.println(Integer.compare(10, 10)); //  0 (相等)
    }

    @Test
    public void testMax() {
        System.out.println(Integer.max(100, 200)); // 200
    }

    @Test
    public void testMin() {
        System.out.println(Integer.min(100, 200)); // 100
    }

    // parseXxx —— 字符串 → 基本类型 ⭐ 最常用！
    // 字符串必须是合法数字，否则抛 NumberFormatException
    @Test
    public void testParseInt() {
        int    num1 = Integer.parseInt("123");      // "123" → 123
        double num2 = Double.parseDouble("3.14");   // "3.14" → 3.14
        long   num3 = Long.parseLong("560");        // "560" → 560L

        System.out.println(num1); // 123
        System.out.println(num2); // 3.14
        System.out.println(num3); // 560
    }

    // 进制转换 —— 十进制 → 其他进制字符串（返回的是 String，不是数字！）
    @Test
    public void testToBinaryString() {
        System.out.println(Integer.toBinaryString(10)); // "1010" (二进制)
    }

    @Test
    public void testToHexString() {
        System.out.println(Integer.toHexString(255)); // "ff" (十六进制)
    }

    @Test
    public void testToOctalString() {
        System.out.println(Integer.toOctalString(64)); // "100" (八进制)
    }

    // compareTo —— 实例方法，比较两个 Integer 对象大小
    // compare vs compareTo：
    //   Integer.compare(10, 20)  → 静态方法，直接传两个int
    //   i1.compareTo(i2)         → 实例方法，需要对象调用
    @Test
    public void testCompareTo() {
        Integer i1 = Integer.valueOf(100);
        Integer i2 = Integer.valueOf(200);
        System.out.println(i1.compareTo(i2)); // -1 (100 < 200)
        System.out.println(i2.compareTo(i1)); //  1 (200 > 100)
        System.out.println(i1.compareTo(i1)); //  0 (相等)
    }

    // equals —— 比较两个包装类对象的【值】是否相等
    // 包装类已重写了 equals() 和 toString()：
    //   equals()   → 比较值（不是地址）✅
    //   toString() → 返回数字字符串（不是类名@哈希）✅
    // ⚠️ 永远用 equals() 比较包装类，不要用 ==
    @Test
    public void testEquals() {
        Integer i1 = Integer.valueOf(100);
        Integer i2 = Integer.valueOf(100);
        Integer i3 = Integer.valueOf(200);

        System.out.println(i1.equals(i2)); // true  → 值相同
        System.out.println(i1.equals(i3)); // false → 值不同
        System.out.println(i1.toString()); // "100" → 重写了，不是内存地址

        Double d1 = Double.valueOf(3.14);
        Double d2 = Double.valueOf(3.14);
        System.out.println(d1.equals(d2));   // true
        System.out.println(d1.toString());   // "3.14"

        Boolean b1 = Boolean.valueOf(true);
        System.out.println(b1.toString());   // "true"
    }

    // intValue() —— 手动拆箱：Integer对象 → int基本类型
    // 自动拆箱：int a = obj;
    // 手动拆箱：int a = obj.intValue();  两者效果完全一样
    @Test
    public void testIntValue() {
        Integer i1 = Integer.valueOf(100);
        int num = i1.intValue(); // 手动拆箱
        System.out.println(num); // 100
    }

    // valueOf(int i) —— 装箱，int → Integer（推荐，有缓存）
    // 缓存范围：-128 ~ 127
    //   范围内：valueOf 返回同一个缓存对象 → == 为 true
    //   范围外：每次创建新对象           → == 为 false
    @Test
    public void testValueOfInt() {
        Integer i1 = Integer.valueOf(100); // 缓存范围内
        Integer i2 = Integer.valueOf(100);
        System.out.println(i1 == i2);     // true（同一个缓存对象）

        Integer i3 = Integer.valueOf(200); // 超出缓存范围
        Integer i4 = Integer.valueOf(200);
        System.out.println(i3 == i4);     // false（不同对象）
        System.out.println(i3.equals(i4));// true ✅ 比值永远用equals！
    }

    // valueOf(String s) —— 字符串 → Integer对象
    // valueOf("123")  → 返回 Integer 对象
    // parseInt("123") → 返回 int 基本类型
    // 需要对象用 valueOf，需要基本类型用 parseInt
    @Test
    public void testValueOfString() {
        Integer i1 = Integer.valueOf("789"); // "789" → Integer(789)
        System.out.println(i1); // 789
        // ⚠️ 传非数字字符串会抛异常！
        // Integer.valueOf("abc"); // NumberFormatException
    }
}
```

## 四、Integer 缓存机制

```java
public class IntegerTest06 {
    public static void main(String[] args) {
        // 案例1：超出缓存范围的数字
        Integer x = 10000;  // 等价于 Integer x = Integer.valueOf(10000);
        Integer y = 10000;
        System.out.println(x == y);  // 输出：false
        // 原因：10000 超出了 Integer 缓存范围 [-128, 127]
        // 所以每次都会创建新的 Integer 对象，x 和 y 指向不同的内存地址
        // 使用 == 比较的是对象引用（内存地址），而不是值，所以返回 false

        // 案例2：在缓存范围内的数字
        Integer a = 127;  // 等价于 Integer a = Integer.valueOf(127);
        Integer b = 127;
        System.out.println(a == b);  // 输出：true
        // 原因：127 在 Integer 缓存范围 [-128, 127] 内
        // Java 提供了整数型常量池（IntegerCache），是一个数组：Integer[] integerCache
        // 数组中存储了 256 个 Integer 对象的引用（-128 到 127）
        // 只要没有超出这个范围的数字，直接从缓存中取同一个对象引用
        // 所以 a 和 b 指向的是同一个对象，== 比较返回 true

        // 案例3：验证 Integer.valueOf() 的行为
        Integer c = Integer.valueOf(127);  // 从缓存获取
        Integer d = Integer.valueOf(127);  // 从缓存获取（同一个对象）
        System.out.println("c == d: " + (c == d));  // 输出：true

        Integer e = Integer.valueOf(128);  // 超出范围，创建新对象
        Integer f = Integer.valueOf(128);  // 再次创建新对象
        System.out.println("e == f: " + (e == f));  // 输出：false

        // 案例4：使用 new 关键字强制创建新对象
        Integer g = new Integer(100);  // 强制创建新对象，不使用缓存
        Integer h = new Integer(100);  // 再次创建新对象
        System.out.println("g == h: " + (g == h));  // 输出：false
        // 原因：new 关键字总是会创建新的对象，不会使用缓存

        // 案例5：正确比较 Integer 对象的方式
        System.out.println("x.equals(y): " + x.equals(y));  // 输出：true
        System.out.println("a.equals(b): " + a.equals(b));  // 输出：true
        // 原因：equals() 比较的是对象的值，而不是内存地址
    }
}
```

**缓存范围总结：**

| 包装类 | 缓存范围 |
| --- | --- |
| `Integer` / `Short` / `Byte` / `Long` | -128 ~ 127 |
| `Character` | 0 ~ 127 |
| `Boolean` | 全部缓存（true/false） |
| `Double` / `Float` | 无缓存 |

## 五、equals / toString / valueOf 对比详解

### 5.1 什么时候用 ==，什么时候用 equals()

```java
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 场景①：基本类型（int/double）→ 用 ==
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
int a = 100;
int b = 100;
System.out.println(a == b);    // true ✅  基本类型没有地址，直接比值

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 场景②：对象比较值 → 用 equals()，不要用 ==
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Integer i1 = Integer.valueOf(200);   // i1 放在内存地址 0x001
Integer i2 = Integer.valueOf(200);   // i2 放在内存地址 0x002
// i1 和 i2 值一样，但是两个不同的对象！
System.out.println(i1 == i2);        // false ❌  地址不同 0x001 ≠ 0x002
System.out.println(i1.equals(i2));   // true  ✅  值相同 200 == 200

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 场景③：判断是否为 null → 只能用 ==
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Integer obj = null;
if (obj == null) {           // ✅ 判断null只能用 ==
    System.out.println("空的");
}
// obj.equals(null)  → 直接报错！NullPointerException ❌
```

### 5.2 什么时候需要重写 equals()

> 核心理解：**JDK 自带的类（Integer、String）已经重写好了，你自己写的类没有重写！**

```java
// 情况①：用 JDK 的包装类/String → 不需要重写，直接用
Integer i1 = Integer.valueOf(100);
Integer i2 = Integer.valueOf(100);
System.out.println(i1.equals(i2));  // true ✅  JDK已经重写好了，直接用

String s1 = new String("hello");
String s2 = new String("hello");
System.out.println(s1.equals(s2));  // true ✅  JDK已经重写好了，直接用

// 情况②：你自己写的 Student 类 → 必须自己重写！
// ❌ 没重写前：内容一样，equals 却返回 false
Student p1 = new Student("张三", 25);  // 地址 0x001
Student p2 = new Student("张三", 25);  // 地址 0x002
System.out.println(p1.equals(p2));     // false ❌  比的是地址！

// ✅ 重写后：按内容比
class Student {
    String name;
    int age;

    @Override
    public boolean equals(Object obj) {
        Student other = (Student) obj;
        return this.name.equals(other.name) && this.age == other.age;
    }
}
// 重写后：
System.out.println(p1.equals(p2));     // true ✅  按内容比了
```

### 5.3 toString()：什么时候省，什么时候不省

> 核心理解：**打印/拼接时自动调用，可以省。要把结果存起来/再处理，不能省。**

```java
Integer i1 = Integer.valueOf(123);

// ✅ 可以省略的场景
// ① println 自动调 toString
System.out.println(i1);              // ✅ 省略，自动输出 123
System.out.println(i1.toString());   // 多此一举，效果一样

// ② 字符串拼接自动调 toString
String msg = "结果是：" + i1;        // ✅ 省略，自动变 "结果是：123"

// ❌ 不能省略的场景
// ③ 要拿到字符串赋值给变量
String str = i1.toString();          // ❌ 不能省！要显式拿到字符串

// ④ 要调字符串的方法
int len = i1.toString().length();    // ❌ 不能省！先转字符串才能调 length()
System.out.println(len);             // 3
```

### 5.4 valueOf()：什么时候省，什么时候不省

> 核心理解：**基本类型→包装类可以省（自动装箱），字符串→包装类不能省。**

```java
// ✅ 可以省略（自动装箱）
Integer i = 100;              // ✅ 等同于 Integer.valueOf(100)，省略了
Double  d = 3.14;             // ✅ 等同于 Double.valueOf(3.14)，省略了

// ❌ 不能省略
// 字符串 → 包装类对象，必须手写！
Integer i2 = Integer.valueOf("123");  // ✅ "123" 变成 Integer(123)
// Integer i2 = "123";               // ❌ 报错！不能直接赋值

// 字符串 → 基本类型，必须用 parseInt！
int num = Integer.parseInt("123");    // ✅ "123" 变成 int 类型的 123
// int num = "123";                  // ❌ 报错！
```

### 5.5 一张表全部搞定

| 方法 | 用它 / 不省略 | 省略 / 不要用 |
| --- | --- | --- |
| `==` | 基本类型比较值；判断对象是否为 null | 比对象的值（用 equals） |
| `equals()` | 两个对象比较值；包装类/String 直接用；自己的类需重写后才能用 | 基本类型不用（用 ==） |
| `toString()` | 要拿字符串去用/存变量时；要调字符串方法前 | println/拼接时省略，自动触发不用写 |
| `valueOf()` | 字符串 → 包装类，不能省 `Integer.valueOf("123")` | 基本类型→包装类可省，`Integer i = 100` 自动装箱 |
| `parseInt()` | 字符串 → 基本类型，不能省 `Integer.parseInt("123")` | 没有可省的情况 |
| 重写 equals | 自己写的类，必须重写！ | JDK 类（Integer/String）已重写好了 |
| 重写 toString | 自己写的类，建议重写！ | JDK 类已重写好了，不用管 |

![综合对比](Attachments/8F6F0BE6-C033-43ED-A741-BBBCF0A7CBE0.png)

**三句话记住核心：**

1. 比对象的值 → 永远用 `equals()`，不要用 `==`（`==` 只用在：基本类型 和 判断 null）
2. `println` / 拼接 → `toString()` 自动触发，省略；要拿字符串结果去用 → 必须手写 `toString()`
3. 基本类型 → 包装类 → 省略（自动装箱）；字符串 → 包装类/基本类型 → 不能省（`valueOf`/`parseInt`）

## 六、String / int / Integer 三种类型互转

> 4 个规律：
> 1. `parseXxx`   : String → 基本类型（例如 int）
> 2. `valueOf`    : 转成"前面那个类"的类型（`Integer.valueOf(...)` → Integer；`String.valueOf(...)` → String）
> 3. `xxxValue()` : 包装类对象 → 基本类型
> 4. `toString()` : 转字符串

```java
public class IntegerTest04 {

    @Test
    public void testStringIntIntegerConvert() {

        // 1) String -> int（最常用）
        // 作用：把"数字字符串"转成可以计算的int
        String s1 = "123";
        int i1 = Integer.parseInt(s1); // parseInt: 解析字符串，返回int
        System.out.println("1.String -> int: " + i1);

        // 2) int -> String（3种常见写法）
        // 作用：把数字变成文本，便于拼接/展示/传输
        int i2 = 100;
        String s2 = i2 + "";              // 写法1：拼接空字符串（简单）
        String s3 = Integer.toString(i2); // 写法2：Integer工具方法
        String s4 = String.valueOf(i2);   // 写法3：String工具方法（推荐，通用）

        System.out.println("2.int -> String(拼接): " + s2);
        System.out.println("2.int -> String(toString): " + s3);
        System.out.println("2.int -> String(valueOf): " + s4);

        // 3) String -> Integer（包装类对象）
        // 作用：得到对象类型，可用于集合/泛型等场景
        String s5 = "123";
        Integer i3 = Integer.valueOf(s5); // 返回Integer对象
        System.out.println("3.String -> Integer: " + i3);

        // 4) Integer -> String（2种写法）
        String s6 = String.valueOf(i3); // 推荐：更安全（即使i3为null也不会NPE）
        String s7 = i3.toString();      // 常用：对象转字符串
        System.out.println("4.Integer -> String(valueOf): " + s6);
        System.out.println("4.Integer -> String(toString): " + s7);

        // 5) int -> Integer（装箱）
        // 作用：基本类型变对象，便于放入集合、调用对象方法
        Integer i4 = Integer.valueOf(100); // 手动装箱（推荐）
        Integer i5 = 600;                  // 自动装箱（最常见）
        System.out.println("5.int -> Integer(valueOf): " + i4);
        System.out.println("5.int -> Integer(自动装箱): " + i5);

        // 6) Integer -> int（拆箱）
        int i6 = i4.intValue(); // 手动拆箱
        int i7 = i5;            // 自动拆箱
        System.out.println("6.Integer -> int(intValue): " + i6);
        System.out.println("6.Integer -> int(自动拆箱): " + i7);
    }

    @Test
    public void testPitfalls() {
        // 坑1：字符串不是纯数字，会抛 NumberFormatException
        // int x = Integer.parseInt("12a");
        // Integer y = Integer.valueOf("abc");

        // 坑2：Integer为null时，toString/自动拆箱会空指针
        Integer n = null;
        // String a = n.toString(); // NullPointerException
        // int b = n;               // NullPointerException（自动拆箱触发）

        // String.valueOf(null) 不抛异常，会得到字符串 "null"
        String safe = String.valueOf(n);
        System.out.println("String.valueOf(null) = " + safe);
    }
}
```

**类型转换决策树：**

```
你要的目标类型是什么？
│
├─ A. 目标是 String（字符串）
│    └─ 用 String.valueOf(x)   ← 万能首选（null也安全）
│       （也可用 x.toString()，但x可能为null会NPE）
│
├─ B. 目标是 基本类型（int/long/double...）
│    ├─ 源是 String  → 用 parseXxx(str)
│    │    例：Integer.parseInt("123") -> int
│    └─ 源是 包装类   → 用 xxxValue() 或自动拆箱
│         例：Integer obj=10; int n=obj; / obj.intValue()
│
└─ C. 目标是 包装类（Integer/Long/Double...）
     └─ 用 Xxx.valueOf(...)
        例：Integer.valueOf("123") / Integer.valueOf(123)
```

## 七、BigInteger — 超大整数

> 应用场景：处理超大整数运算（如金融计算、密码学、科学计算）  
> `long` 最大值：9,223,372,036,854,775,807（约 922 亿亿）  
> `BigInteger` 理论无上限，只受内存限制

```java
import java.math.BigInteger;

public class BigNumTest01 {
    public static void main(String[] args) {

        // 普通 long 类型无法存储超大数字
        // long num = 9999999999999999999999L; // 编译报错：数字太大

        // 创建 BigInteger 对象（必须使用字符串构造函数，因为数字已超出 long 范围）
        BigInteger num1 = new BigInteger("9999999999999999999999");
        System.out.println("原始大数：" + num1);

        BigInteger num2 = new BigInteger("1");

        // 四则运算（BigInteger 是不可变对象，运算后会返回新对象，原对象不变）

        // 加法：num1 + num2
        BigInteger result1 = num1.add(num2);
        System.out.println("加法结果：" + result1);  // 10000000000000000000000

        // 减法：num1 - num2
        BigInteger result2 = num1.subtract(num2);
        System.out.println("减法结果：" + result2);  // 9999999999999999999998

        // 乘法：num1 * num2
        BigInteger result3 = num1.multiply(num2);
        System.out.println("乘法结果：" + result3);

        // 除法：num1 / num2（整数除法，向下取整）
        BigInteger result4 = num1.divide(num2);
        System.out.println("除法结果：" + result4);

        // 幂运算：2^3 = 8
        BigInteger bigInteger = new BigInteger("2");
        BigInteger pow = bigInteger.pow(3);
        System.out.println("幂运算结果：2^3 = " + pow);

        // 开平方运算（返回整数平方根，向下取整）
        BigInteger bigInteger1 = new BigInteger("9");
        BigInteger sqrt = bigInteger1.sqrt();
        System.out.println("开平方结果：√9 = " + sqrt);

        BigInteger bigInteger2 = new BigInteger("10");
        System.out.println("开平方结果：√10 = " + bigInteger2.sqrt() + " (实际是3.162...,向下取整为3)");
    }
}
```

**注意：**

```java
// ❌ 不存在 int 构造器！
new BigInteger(9);  // 编译报错！必须传字符串

// ✅ 普通数字用 valueOf
BigInteger b = BigInteger.valueOf(100);
```

## 八、BigDecimal — 精确小数

> 应用场景：金融计算（货币、利息、税率）、需要高精度计算的场景、对舍入误差零容忍的业务  
> **涉及钱的计算绝对不能用 `double`，必须用 `BigDecimal`！**

```java
import java.math.BigDecimal;

public class BigNumTest02 {
    public static void main(String[] args) {
        // 基本四则运算
        BigDecimal num1 = new BigDecimal("10");
        BigDecimal num2 = new BigDecimal("2");

        System.out.println("加法：" + num1.add(num2));       // 12
        System.out.println("减法：" + num1.subtract(num2));  // 8
        System.out.println("乘法：" + num1.multiply(num2));  // 20
        System.out.println("除法：" + num1.divide(num2));    // 5

        // 小数点移动
        BigDecimal num3 = new BigDecimal("123456789.123456789");
        System.out.println("左移3位：" + num3.movePointLeft(3));   // 123456.789123456789
        System.out.println("右移3位：" + num3.movePointRight(3));  // 123456789123.456789

        // ❌ 重要陷阱：不要用 double 构造
        BigDecimal num4 = new BigDecimal(0.1);  // 错误写法
        System.out.println("用 double 构造 0.1：" + num4);  // 0.10000000000000000555...

        BigDecimal num5 = new BigDecimal("0.1");  // 正确写法
        System.out.println("用 String 构造 0.1：" + num5);  // 0.1

        // 除法注意事项
        BigDecimal num6 = new BigDecimal("10");
        BigDecimal num7 = new BigDecimal("3");

        // 错误：10/3 除不尽会抛异常 ArithmeticException
        // System.out.println(num6.divide(num7));

        // 正确：指定精度和小数位数，以及舍入模式（ROUND_HALF_UP = 四舍五入）
        System.out.println("10/3 四舍五入保留2位：" +
            num6.divide(num7, 2, BigDecimal.ROUND_HALF_UP));
    }
}
```

**三个必须记住的坑：**

```java
new BigDecimal(0.1)          // ❌ 精度丢失
new BigDecimal("0.1")        // ✅ 正确，必须传字符串

price.divide(count)          // ❌ 除不尽 → 抛异常！
price.divide(count, 2, RoundingMode.HALF_UP)  // ✅ 必须指定精度

BigDecimal.valueOf(0.1)      // ✅ 手上只有 double 变量时用这个
```

## 九、DecimalFormat — 数字格式化

> 应用场景：数字千分位展示（财务报表、价格显示）、固定小数位数格式化、百分比格式化

**格式规则：**

| 符号 | 含义 |
| --- | --- |
| `#` | 数字位，不够不补 0 |
| `0` | 数字位，不够补 0 |
| `,` | 千分位分隔符 |
| `.` | 小数点 |

```java
import java.text.DecimalFormat;

public class DecimalFormatTest {
    public static void main(String[] args) {

        // 千分位格式化
        // 规则：整数部分每3位加逗号，小数部分最多保留2位（不够不显示）
        DecimalFormat df = new DecimalFormat("###,###.##");
        String s = df.format(12345678.123);
        System.out.println("千分位格式：" + s);  // 输出："12,345,678.12"

        // 固定小数位数
        // 规则：小数部分固定4位，不够补0，超过截断
        DecimalFormat df2 = new DecimalFormat("###,###.0000");
        String s2 = df2.format(12345678.123);
        System.out.println("固定4位小数：" + s2);  // 输出："12,345,678.1230"

        // 场景A：金额格式化（保留2位，不够补0）
        DecimalFormat moneyFormat = new DecimalFormat("¥###,###.00");
        System.out.println("金额格式：" + moneyFormat.format(999.9));  // ¥999.90

        // 场景B：百分比格式化
        DecimalFormat percentFormat = new DecimalFormat("#0.00%");
        System.out.println("百分比：" + percentFormat.format(0.8567));  // 85.67%

        // 场景C：科学计数法
        DecimalFormat scientificFormat = new DecimalFormat("0.00E0");
        System.out.println("科学计数法：" + scientificFormat.format(12345678.0));  // 1.23E7
    }
}
```

**byte** — 微小整数

```java
byte b = 100;
```

```java
int age = 25;
int max = Integer.MAX_VALUE;  // 2147483647
System.out.println(max + 1);  // -2147483648 ← 溢出，不报错！
```

**long** — 大整数 ⭐

```java
long phone = 13812345678L;    // ⚠️ 必须加 L
long time  = System.currentTimeMillis();
long order = 202412311234567890L;
// ✅ 手机号、时间戳、订单号、金额（单位：分）
// ⚠️ 字面量必须加 L，不加编译器当 int 处理报错
```

**double** — 双精度小数 ⭐

```java
double pi    = 3.14159;
```

**char** — 单个字符 ⭐

```java
char c1 = 'A';        // 单引号！

System.out.println(c3);  // 输出：A

// char 可以做运算
char c4 = 'A';

System.out.println((char)(c4 + 1));  // 输出：B
// ⚠️ 是单引号 ' '，字符串是双引号 " "
// ⚠️ char 参与运算会自动升级成 int
```

**boolean** — 布尔值 ⭐

```java
boolean isLogin = true;
boolean hasPermission = false;
```
